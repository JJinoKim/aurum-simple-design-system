Aurum Design System · npm package

Ship Aurum to npm.

Tokens, themes, and components are already authored as React + TypeScript + styled-components. The package layer wraps them in a tsup build and a GitHub Actions release pipeline, so a single git push --tags ships a versioned bundle to the registry — with light/dark theming, type declarations, and three subpath entries.

react ≥ 18 styled-components 6 typescript 5 esm + cjs + .d.ts provenance signed

Three entry points

Subpath exports keep tokens reachable without pulling React.

SpecifierWhat you getFormat
aurum-design-system Tokens + ThemeProvider + all web components esm · cjs
…/tokens Raw scales + light/dark themes (no React) esm · cjs
…/native React Native entry (resolves *.native.tsx) esm · cjs

Each entry ships sourcemaps and a .d.ts file. The package is marked "sideEffects": false so consumers tree-shake unused exports.

02 · Consuming it

Install, wrap, render.

Install

Peer deps stay external — your app's React/styled-components are deduped.

$ npm install aurum-design-system styled-components react react-dom

Wrap your app

ThemeProvider takes initialMode and exposes useThemeMode().toggle.

import { ThemeProvider, GlobalStyle } from 'aurum-design-system';

export function App() {
  return (
    <ThemeProvider initialMode="dark">
      <GlobalStyle />
      <YourRoutes />
    </ThemeProvider>
  );
}

Use the components

Every component reads tokens through styled(theme => …); mode swaps at runtime.

import {
  Button, Input, Card,
  useThemeMode,
} from 'aurum-design-system';

export function SignIn() {
  const { mode, toggle } = useThemeMode();
  return (
    <Card>
      <Card.Header title={"Sign in"} />
      <Card.Body>
        <Input placeholder="you@aurum.dev" />
        <Button variant="primary">Continue</Button>
        <Button variant="ghost" onClick={toggle}>
          {mode === 'dark' ? 'Light' : 'Dark'} mode
        </Button>
      </Card.Body>
    </Card>
  );
}

04 · GitHub Actions

Two workflows. Tag → publish.

ci.yml — every push & PR

Matrix on Node 18 + 20. Fails fast on lint, type, or build errors before merge.

on: { push: { branches: [main] }, pull_request: }
jobs:
  verify:
    strategy: { matrix: { node: [18, 20] } }
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: ${{ matrix.node }}, cache: npm }
      - run: npm ci
      - run: npm run lint
      - run: npm run typecheck
      - run: npm run build
      - run: npm pack --dry-run

release.yml — on v*.*.* tag

Verifies the tag matches package.json, rebuilds, publishes with provenance, drafts a GitHub Release.

on: { push: { tags: ['v*.*.*'] } }
jobs:
  publish:
    permissions: { id-token: write, contents: write }
    steps:
      - run: # tag === package.version ?
      - run: npm ci && npm run build
      - run: npm publish --access public --provenance
        env: { NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }} }
      - uses: softprops/action-gh-release@v2

Release flow — what the maintainer types

All version juggling stays in package.json; the workflow refuses to publish if the tag drifts.

# 1. bump the version (also writes the git tag locally)
$ npm version minor   # 0.1.0 → 0.2.0

# 2. push commit + tag — release.yml takes over from here
$ git push origin main --follow-tags

# 3. CI runs lint → typecheck → build → npm publish --provenance
#    GitHub Release is drafted with auto-generated notes.

05 · One-time setup

Three things to wire before the first publish.

NPM_TOKEN

npmjs.com → Access Tokens → Granular automation token scoped to this package. Add it in the repo's Settings → Secrets → Actions.

npm provenance

The release job sets id-token: write and publishes with --provenance, so the published tarball is signed by GitHub OIDC. Nothing for you to configure.

Repository fields

Update repository.url, homepage, and bugs in package.json to your real org/repo before the first tag.
